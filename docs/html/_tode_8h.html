Detailed Description

Tode Class. 

Two different types of Tode will exist.
    

  • ThisTode
 EEPROM Configuration with accessable Remote Control at 0x0000
 Device Settings like IO-Pin stored in LOCAL SYSTEM SETTING EEPROM space. (Not Control Space)
 All Device Item Values will be gotten from HW-Control.  Those Values will need RF-requested
    • 

  • Remote Todes
 All Value SET/GET will be acheived through RF requests
    

     System Settings will be saved at the tail of EEPROM space.
 ThisTode will be the first Tode at the start of EEPROM space.
 Subsequent Remote Tode configuration will be stored after ThisTode from start.
 We will no longer use Tode-UID but instead use RF-Addr + 15 Arduino ID from Peer.
 Pinging will occur on FFFF but be filtered by the 1-nibble Arduino ID.
