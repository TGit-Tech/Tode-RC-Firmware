\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_tode}{Tode}} Class. 


\begin{DoxyPre}
Two different types of \mbox{\hyperlink{class_tode}{Tode}} will exist.
\begin{DoxyItemize}
\item ThisTode
 EEPROM Configuration with accessable Remote Control at 0x0000
 \mbox{\hyperlink{class_device}{Device}} Settings like IO-\/Pin stored in LOCAL SYSTEM SETTING EEPROM space. (Not Control Space)
 All \mbox{\hyperlink{class_device}{Device}} Item Values will be gotten from HW-\/Control.  Those Values will need RF-\/requested
\item Remote Todes
 All Value SET/GET will be acheived through RF requests

 System Settings will be saved at the tail of EEPROM space.
 ThisTode will be the first \mbox{\hyperlink{class_tode}{Tode}} at the start of EEPROM space.
 Subsequent Remote \mbox{\hyperlink{class_tode}{Tode}} configuration will be stored after ThisTode from start.
 We will no longer use Tode-\/UID but instead use RF-\/Addr + 15 Arduino ID from Peer.
 Pinging will occur on FFFF but be filtered by the 1-\/nibble Arduino ID.
